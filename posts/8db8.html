GitHub·图床教程

发表于2022-11-06|更新于2022-11-06|代码
|字数总计:607|阅读时长:1分钟|阅读量:

注册Github

首先你得有一个Github账号，去 http://github.com 注册(注册方法这里我就不多赘述了，无非就是用户名密码)。

新建图床仓库

注册好后点击头像，选择Your repositories。

点击右上角绿色NEW按钮

名字随意，注意选择仓库类型是公共使用的，选public,其它不用管,直接点击创建。

生成token操作你的仓库

具体步骤为settings→Developer settings→personal access tokens→Tokens→Generate new token(classic),令牌和令牌过期时间自己设置。
注意之后一定要勾选repo,其他无所谓，之后点击生成。注意之后生成的token只会出现那每一次，建议把它复制到其他地方防止丢失。

配置


根据仓库的 url 地址去填写你自己的配置：
比如上图中的 repourl 是: github.com/CarGuo/GSYVideoPlayer
那么填入以下配置
用户名：CarGuo
仓库名：GSYVideoPlayer

至此配置完毕，已经可以使用了。当你上传的时候，你会发现你的仓库里也会增加新的图片了。

上传图片

通过上面的步骤，我们的图床时搭建好了，但是通过传统的方法向 Github 上传图片太麻烦了，这里我们推荐使用一个开源图床工具 PicGo 来作为我们的图片上传工具；
PicGo官网 https://molunerfinn.com/PicGo/ 下滑找到自己可用的版本进行下载。、
安装好后打开样式

点击图床设置→点击github→如图填写即可
。
至此，图床已经可以步入使用了，上传图片，通过上传区上传即可（Ctrl V 或者将图片拖拽都可以），也可以通过快捷键的方式上传（默认上传键为 Ctrl + Shift + P）；

用 jsDelivr 给自己的 github图床进行加速

大家可能会发现，我们上传到 Github 的图片有时候访问太慢了，有时候甚至直接加载不出来，这时候就要用到 jsDelivr 进行免费加速，而设置的方法也很简单，只需要在我们 PicGo 图床配置中添加如下自定义域名即可；
https://cdn.jsdelivr.net/gh/用户名/仓库名
比如我的就是 https://cdn.jsdelivr.net/gh/cunyu1943/blog-imgs)

到目前为止，jsDeliver加速貌似已经没用

推荐使用 http://cdn.staticaly.com/gh/用户名/仓库名@master

至此，github图床教程结束。
